# Scientific Validation Subjects

This directory is for future subject-registration artifacts only.

A future scientific-validation subject is not eligible to start unless its registration proves:

- the first logic-changing subject commit happened after the protocol was frozen
- the subject is not metadata-only
- the benchmark corpus was frozen before subject logic started
- the benchmark corpus and runner are protocol-owned, not subject-test-owned

The registration artifact should follow:

- [subject-registration-template-v1.toml](/Users/spson/Projects/Claude/bolt-v2/.worktrees/issue-208-validation-protocol/docs/mechanical-process-package/validation/subjects/subject-registration-template-v1.toml:1)

It is an entry artifact, not a verdict artifact.
