# Data Model: CI Nextest Artifact Cache

## NextestArtifactCache

- **Represents**: The rust-cache configuration for the sharded `test` job.
- **Fields**:
  - `workspace_root`: `.`
  - `managed_target_dir_relative`: relative path from `GITHUB_WORKSPACE` to the managed Rust target directory
  - `cache_targets`: true
  - `cache_workspace_crates`: true
  - `cache_on_failure`: true
  - `add_rust_environment_hash_key`: true
  - `key`: bounded shard key
- **Validation**: Verifier fails if workspace target mapping, workspace crate preservation, rust-environment hashing, shard keying, or managed target-dir opt-in is missing.

## ShardCacheKey

- **Represents**: The bounded per-shard namespace layered onto rust-cache's automatic key.
- **Fields**:
  - `prefix`: `nextest-v3`
  - `shard`: `${{ matrix.shard }}`
  - `total`: `4`
  - `rust_environment_hash`: generated by rust-cache
- **Validation**: Must include `matrix.shard` and `of-4`; must not include `github.sha`.

## RustEnvironmentHash

- **Represents**: rust-cache's automatic invalidation source for real Rust inputs.
- **Inputs**:
  - Rust compiler release, host, and commit hash
  - Compiler-related environment variables with `CARGO`, `CC`, `CFLAGS`, `CXX`, `CMAKE`, and `RUST` prefixes
  - Cargo manifests and lockfile
  - `rust-toolchain` / `rust-toolchain.toml`
  - `.cargo/config.toml` when present
- **Validation**: Workflow must explicitly keep `add-rust-environment-hash-key: "true"` for the `test` cache step.

## WarmRerunEvidence

- **Represents**: Completion evidence required by #195.
- **Fields**:
  - `cold_run_id`
  - `warm_run_id`
  - `head_sha`
  - `test_shard_job_ids`
  - `cache_hit_lines`
  - `restored_key_lines`
  - `archive_size_lines`
  - `compile_log_findings`
  - `cold_warm_timing_comparison`
- **Validation**: Cannot be marked complete until exact GitHub Actions data exists.

## CacheGrowthEvidence

- **Represents**: Cache-size and pruning/eviction evidence.
- **Fields**:
  - `before_archive_size_bytes`
  - `after_archive_size_bytes_by_shard`
  - `rust_cache_cleanup_lines`
  - `github_cache_list_entries`
  - `eviction_or_thrashing_notes`
- **Validation**: Evidence must compare before/after and explain bounded key shape.

## GateInvariant

- **Represents**: Required CI semantics independent of cache state.
- **Fields**:
  - `test_needs_source_fence`
  - `gate_needs_test`
  - `gate_accepts_only_test_success`
  - `test_execution_command`
- **Validation**: Cache miss, stale cache, or cache failure must still lead to normal `just test -- --partition count:${{ matrix.shard }}/4` execution.
